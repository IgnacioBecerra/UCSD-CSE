An Introduction to Unix

Unix is a family of operating systems that share a common philosophy and follow a set of standards. The operating system with the largest install base that meets these standards (the Single UNIX Specifications) is Apple's OS X. Linux is a popular clone of Unix that is largely compliant with these standards and follows the same philosophies as Unix.

Nearly all of what will be learned in this course will be applicable to both OS X and all types of Linux systems although the lab computers are running CentOS a distribution of Linux. One characteristic of the Unix philosophy is the idea that programs should be small and focused, accomplishing a single task well, then through a command-line interpreter (a shell) we can weave these programs together to accomplish more complex tasks.

"...the idea that the power of a system comes more from the relationships among programs than from the programs themselves". - The Unix Programming Environment

These philosophies make the system robust, flexible and extensible while reducing complexity of any single program. To begin we'll familiarize ourselves with a command prompt, and learn some basic commands to help give us the context of our shell and the ability to perform basic file operations that most of us are probably used to doing in a graphical program such as Finder or Windows Explorer.

An Introduction to Vim

In the second portion of the lab we'll be going over the basic usage of a text editing program named Vim. Vim stands for Vi Improved, Vi being the predecessor to Vim. To continue, Vi is a shortening of Visual as it is a visual editor. While we won't go into depth of the differences between Vi and Vim, the basic usage for both is the same and nearly all of what will be learned today is applicable to both. Before we go into the basic usage of Vim, let's discuss some of the motivations for using it. Basic knowledge and usage of Vim is a must know in the Unix world as it has become a standard text editor across many systems and knowing how to use Vi or Vim will come in handy if it's the only text editor available to you. Vim can run directly in the terminal without the need for a graphical interface we can use it while remotely logging into other systems where our editor choices may be limited. Even more important is Vim's power as a text editor, allowing a user to describe changes to a pieces of text as patterns and replicate the changes many times saving a lot of time.

Goal: Opening and Closing Vim

In comparison to other text editors you may be used to, Vim does not rely on the use of a mouse, and everything is done via the keyboard. While there are versions of Vim that have graphical interfaces (GVim, Macvim, etc.), they will not be the focus. That being said, using Vim is likely going to be very different from what you have used in the past, so be prepared to try something new!

Before you begin using Vim, ensure that you're in your Lab1 directory. To open Vim for the first time, simply issue the vim command.

Question 26: According to the text in the middle of the screen, how can you exit Vim?

Now that we've opened Vim, how do we close it and get back to bash? First we need to understand the concept of Vim's modes. Vim separates tasks into separate modes to make full use of the keyboard, if all letters were bound to insert that letter into the document there wouldn't be any keys left for anything else! Over the course of the lab we'll go over four of Vim's modes: normal or command mode, insert mode, visual mode, and the command-line mode. By default Vim opens up in the normal mode, which is used for navigation and the manipulation of text by issuing commands to go with those movements. Generally speaking, you can return to the normal mode by hitting the Esc key. Next is the command-line mode, which is used for typing in various Vim commands. Please note that the command-line mode is not the same as the command line in bash. To enter the command-line mode hit the : (colon) key, followed by the command to be executed which will be displayed in the lower left hand corner. For now, we want to quit vim so we'll issue the quit editor command. The full sequence of keys to to exit is:

or for short:

:q

Great! We've gotten in and out of Vim. Next let's actually use it to add some text to a file.

Goal: Navigation and movement commands

Vim uses a navigation scheme that will likely be new to you. That's right! We haven't talked about saving files with Vim. It's good to get into a habit of saving often! If Vim has detected any changes to the buffer since opening it, it won't allow you to quit. You must either save the file or add an ! to the end of the command telling Vim to explicitly quit without changes. We want to save so let's issue the write command:
